# Release Readiness Checklist

_Last updated: 2026-04-15_

## Package trust checklist

- [x] Package exposes explicit `main`, `module`, and `types` entries.
- [x] Published files include distribution assets, README, and license.
- [x] `exports` map includes `types` for tooling compatibility.
- [x] Repository metadata (`repository`, `homepage`, `bugs`) is present.
- [x] SPDX license field and root `LICENSE` file are present.

## Documentation consistency checklist

- [x] Root README links to current docs and examples.
- [x] Docs index links only to current, existing files.
- [x] Workflow messaging is consistent across README/docs/examples.
- [x] Legacy planning docs are clearly marked as historical context.

## Launch polish checklist

- [x] Roadmap document includes current priorities and date.
- [x] Release notes template exists for first tagged release.
- [x] Label taxonomy documented for issue hygiene.
